打卡方式

三种途径，随时随地完成早起 / 早睡 / 运动打卡。

🌐
Portal 页面
Web UI
浏览器打开 Portal 首页，一键打卡。最直观，支持三种打卡类型。
界面预览
☀️ 早起 🌙 早睡 🏃 运动
☀️ 今日尚未打卡
🌙 今日尚未打卡
🏃 今日尚未打卡
使用方式
1
打开 junethebest.cn
2
登录（密码认证）
3
选择打卡类型，点击按钮
✓
自动记录时间，显示连续天数
特点
  • 可视化热力图，一眼看月度表现
  • 三种打卡类型切换（早起/早睡/运动）
  • 统计数据实时展示
  • 需要登录认证
🔌
Webhook API
API
HTTP POST 调用，无需浏览器。适合脚本、定时任务、其他服务集成。
请求示例
POST /api/webhook/checkin X-API-Key: your-webhook-key Content-Type: application/json { "wake_time": "05:30", "source": "ha-automation" }
响应
{ "ok": true, "date": "2026-05-11", "wake_time": "05:30", "pass": true, "streak": 7, "source": "ha-automation" }
使用场景
  • curl / Python / 任意 HTTP 客户端
  • 不传 body 默认用当前时间
  • X-API-Key 认证，无需登录
  • source 字段追踪来源
curl 一行命令
curl -X POST https://junethebest.cn/api/webhook/checkin \ -H "X-API-Key: your-key" \ -H "Content-Type: application/json" \ -d '{"wake_time":"05:30","source":"curl"}'
⚙️
自动化触发
HA / iOS
Home Assistant 自动化或 Apple 快捷指令，无感打卡。
方式 A: Home Assistant
1
手机离开床（传感器触发）
2
HA 自动化执行 webhook
3
记录起床时间 + 连续天数
# HA configuration.yaml automation: - alias: "自动早起打卡" trigger: - platform: state entity_id: binary_sensor.bed_occupancy to: "off" action: - service: rest_command.checkin rest_command: checkin: url: https://junethebest.cn/api/webhook/checkin method: POST headers: X-API-Key: your-webhook-key content_type: "application/json"
方式 B: Apple 快捷指令
1
Siri / NFC / Widget 触发
2
快捷指令发送 POST 请求
✓
收到通知确认打卡结果
快捷指令配置: 操作: 获取 URL 内容 URL: https://junethebest.cn/api/webhook/checkin 方法: POST Headers: X-API-Key: your-webhook-key Body (JSON): {"source": "ios-shortcut"}
特点
  • 完全无感，起床自动打卡
  • 支持 NFC 标签贴床头触发
  • iOS Widget 一键点击
  • HA 可联动灯光/窗帘
三种方式数据互通 · 共享同一数据库 · 连续天数实时同步
