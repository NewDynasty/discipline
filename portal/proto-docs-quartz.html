pitfall z.ai glm api
创建于 2026-05-02 · 最后修改 2 小时前 · 编辑此页

问题背景 #

使用 z.ai GLM 系列模型 API 时，遇到了一系列非预期行为。记录于此，避免重复踩坑。

⚠️
本文档基于 2026 年 5 月的实际测试。 z.ai 可能已修复部分问题，请以最新 API 文档为准。

坑 1：Smart Routing 静默降级 #

当启用 smart_routing 参数时，API 会静默将复杂请求降级到 GLM-4.5-Air 而不通知调用方。

💡
关键教训：永远不要信任 API 会按请求的模型返回结果，务必检查 x-model-used 响应头。

坑 2：Token 计数不一致 #

z.ai 的 token 计数与 OpenAI 标准存在差异：

指标OpenAI 标准z.ai 实际差异
prompt_tokens1,2471,583+27%
completion_tokens512641+25%

坑 3：并发限制无文档 #

官方文档写着 Max Plan 支持"高并发"，但实际并发限制 = 3，文档里没写。

✅
解决方案：控制并发 ≤ 3，或在客户端加请求队列。
